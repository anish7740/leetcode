# DSA Study Plan — NeetCode 150 (by pattern)

The order to grind, why each pattern matters, and a checklist. Goal: one problem/day, Blind 75 first, then the rest of NeetCode 150. Target: interview-ready for Summer 2027 (Google STEP, Capital One, Microsoft).

## Order of attack (do patterns top-to-bottom)
1. **Arrays & Hashing** — the foundation; most warmups live here. (Two Sum, Group Anagrams, Top K)
2. **Two Pointers** — sorted-array / palindrome tricks. (Valid Palindrome, 3Sum)
3. **Sliding Window** — substring/subarray optimization. (Best Time to Buy/Sell, Longest Substring)
4. **Stack** — parsing, monotonic stacks. (Valid Parentheses, Daily Temperatures)
5. **Binary Search** — search space reduction. (Search Rotated Array, Koko Eating Bananas)
6. **Linked List** — pointer manipulation. (Reverse List, Merge Two Lists, Detect Cycle)
7. **Trees** — DFS/BFS, recursion. (Invert Tree, Level Order, Validate BST)
8. **Tries** — prefix problems. (Implement Trie, Word Search II)
9. **Heaps / Priority Queue** — top-K, scheduling. (Kth Largest, Task Scheduler)
10. **Backtracking** — combinations/permutations. (Subsets, Combination Sum, Word Search)
11. **Graphs** — BFS/DFS, union-find. (Number of Islands, Course Schedule, Clone Graph)
12. **Dynamic Programming** — 1D then 2D. (Climbing Stairs, House Robber, Coin Change, LCS)
13. **Intervals / Greedy / Bit** — the finishers.

## Daily workflow
1. Pick the next problem in the current pattern.
2. Solve it in `solutions/NNNN-name.py` (and re-solve key ones in `.cpp` to make "C++" real).
3. Add a top comment: pattern, time/space complexity, and the key insight.
4. Tick the box below + add a row to the README table.
5. Commit: `git commit -m "NNNN. Problem Name (pattern, O(n))"`.
